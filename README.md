# OPTEX — Optimal Execution & Market Impact Intelligence Platform

## Problem
Large orders move markets. OPTEX finds the execution schedule that minimizes implementation shortfall using reinforcement learning and classical models.

## Architecture
```
User -> Dash UI (pages) -> Flask server
                     |-> RL inference (stable-baselines3)
                     |-> Backtester (Monte Carlo)
                     |-> Data layer (Polars, sample/Kaggle LOB)
Docker -> Gunicorn -> Dash server -> WebSocket (yfinanace live)
```

## Quick start
```bash
git clone <repo>
cd optex
docker-compose up --build
# open http://localhost:8050
```

## Data
- Default: synthetic LOB generated by `scripts/generate_sample_data.py` (OU mid, Pareto depth, power-law trades)
- Optional: download Kaggle Optiver dataset with `scripts/download_data.py` (requires kaggle CLI)

## Train your own agents
```bash
python src/training/train_agents.py --total_timesteps 200000
```
Models saved to `models_saved/`.

## Deploy to Render
1. Push repo to GitHub
2. Create Render Web Service, select `render.yaml`
3. Deploy — health endpoint `/health`

## API reference
- Simulation endpoint: Dash callbacks accept JSON via browser; core simulation uses `src/environment/lob_env.py` and `src/models/impact_models.py`.

## Strategies
- **TWAP**: equal slices
- **VWAP**: volume profile weighting
- **Almgren–Chriss**: closed-form optimal trajectory
- **PPO/SAC/TD3**: RL agents on LOB features

## Metrics
- IS bps, IS Sharpe, win rate vs TWAP, CVaR 95, completion rate, Gini of schedule, Wilcoxon p-values
